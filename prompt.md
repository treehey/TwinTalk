# 角色设定
你是一个资深的前端 React 开发工程师，精通移动端 H5 响应式开发和 React 状态管理。

# 核心任务
目前我们的 React 项目中有一个名为 “Advisor” 的底部导航栏目（或入口）。现在需要你将其整体替换为 “Post（动态/社区）” 栏目。用户在移动端点击 “Post” 后，将进入一个全新的信息流（Feed）交互界面，允许用户发帖、浏览动态并进行基础互动。

# 参考规范
请严格参考本项目中 `react-app.js` 的代码风格、UI 组件结构、CSS 变量（如 `var(--c-bg)`, `var(--c-text-primary)` 等）以及交互逻辑来进行开发，保持整体视觉和交互的一致性。

# 具体开发需求

## 1. 导航与入口改造
* 全局搜索原有的 "Advisor" 入口（可能在 BottomNav、TabBar 或 Drawer 中）。
* 将文本标签从 "Advisor" 更改为 "动态" 或 "Post"。
* 替换对应的 Icon（建议使用类似于编辑、发送或星球社区的图标）。
* 修改对应的路由或状态切换逻辑，使其点击后渲染全新的 `<PostView />` 组件。

## 2. 开发新的 `<PostView />` 组件
请创建一个包含以下功能模块的移动端响应式页面：

### A. 顶部导航/标题栏 (Header)
* 包含页面标题（如 "社区动态"）。
* 风格需与现有 App 的 Header 保持一致（参考 `react-app.js`）。

### B. 发帖区域 (Create Post)
* 一个多行文本输入框（Textarea），支持自动扩展高度，带有 Placeholder（如 "分享你的最新动态..."）。
* 一个 "发布" (Post/Send) 按钮。输入框为空时按钮应处于禁用状态。
* (可选) 支持基础的 Emoji 插入，复用项目中已有的表情选取逻辑。

### C. 动态信息流列表 (Feed List)
* 渲染一个动态卡片列表。如果没有数据，请展示优雅的空状态（Empty State）。
* **单条动态卡片 (Post Card) 包含：**
  * **发帖人信息**：头像（Avatar）、昵称（Nickname）、发布时间（需要包含类似于 "刚刚", "x分钟前" 的相对时间格式化逻辑）。
  * **正文内容**：展示用户发布的文本，支持长文本折叠/展开。
  * **互动操作区 (Actions)**：
    * 点赞按钮（显示点赞数，点击切换状态）。
    * 评论按钮（显示评论数）。
    * 分享/更多按钮。

## 3. 状态管理与 API 接口预留
* 在组件内部使用 `useState` 和 `useEffect` 管理列表数据、输入框内容、加载状态（Loading）和提交通知（Toast）。
* 编写标准的 API 调用占位函数（如 `fetchPosts()`, `submitPost(content)`, `toggleLike(postId)`），并用 Mock 数据先行填充，以便于后续与后端进行联调。

## 4. 工程规范
* 请提供完整的拆分组件代码（例如可以将卡片拆分为 `<PostCard />`，发帖区拆分为 `<PostEditor />`）。
* 确保代码在移动端设备上的滚动体验流畅（考虑 `overflow-y: auto` 和适宜的 `padding-bottom` 防底部导航遮挡）。
* 直接输出可运行的 React 代码。
